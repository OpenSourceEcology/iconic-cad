ICONIC CAD OPEN SOURCE ECOLOGY

A house you can read.

Iconic CAD compiles a plain text file into framing drawings, a priced cut list, and a printable build book for the Seed Eco-Home. Change the file, and every drawing, price, and instruction changes with it. Try the editor in your browser.

One entry in wall_instances.yaml and the panel it compiles to. Dimensions in inches. Every member here — plates, kings, jacks, cripples, sill, header — is enumerated by the same function that prices the cut list.

One file, three outputs

.FCSTDDrawings Real FreeCAD assemblies, compiled offline or exported straight from the browser. No CAD install needed to lay out a floor plan.
.CSV / $Priced cut list Lumber, sheathing, and fasteners counted from the framing itself, priced from a catalog. Change a wall, the bill changes.
.PDFBuild book Fabrication cards with elevations, cut lengths, procedure, and QC checks — a packet a build crew can hold.

Where it stands

We shipped v2 in July 2026. Collin DeSantis built v1 in spring 2026: fifteen wall and window modules, the drag-and-snap layout editor, and two independent CAD compilers held equal by parity tests. v2 puts a validated library underneath all of it. Each of the fifteen modules is now a library entry with its own parameters, compiler, and geometric expectations, checked on every change — and every drawing, cut list, and browser asset is baked only from entries that pass. The editor now runs in the browser, no install. The validators also caught framing defects in our designs; those findings are tracked openly. To keep the specifications this compiler reads — they live on the OSE wiki — accurate and current, we built wikibot, an editor that drafts wiki improvements for people to review.

The part library

In July 2026 we developed vcs-library, a part library for the Village Construction Set 12-foot modules, seeded with Catarina’s rafters, wall modules, ceiling, and cabin assemblies. Every entry has the same anatomy, following the wiki’s Schema Canon: a schema (the parameters, plain data), a compiler (the geometry), and two validators that judge it. Code validation checks the parameters — units, ranges, required fields — in seconds, on every change. Output validation compiles the entry under FreeCAD and inspects the result: are all the members there, do any occupy the same space, does the whole thing fit its declared envelope. From the same pipeline, every entry also gets a generated bill of materials and fabrication drawing — documents that regenerate whenever the design changes, so they cannot drift from it. For library authors who work in FreeCAD, the library workbench brings the whole loop — browse, compile, edit parameters, validate — into FreeCAD itself.

REV — LIBRARY

Twelve entries seeded across three layers — parts, modules, assemblies — with both validators running on every change. Browse them at the library site — every entry gets a page with its parameters, 3D model, fabrication drawing, bill of materials, and validation status, regenerated from the library on every change. Next layers up: whole structures, then further libraries. The library ontology exists so many people can each own, fork, and validate their own libraries without stepping on each other.

Where it goes

The Seed Eco-Home needs hundreds of module types — walls today; roofs, floors, stairs, wiring, and plumbing to come. The plan is not to draw them all by hand. Because a module is a few lines of text judged by automated tests, drafting new ones is work AI agents can do and people can verify. The OSE wiki holds the specifications; this compiler turns them into things a crew can build. That is the bet: documentation that regenerates itself is how a village-scale construction set stays buildable by strangers.

Downstream, the compiled house — drawings, cut lists, build book — folds into a refreshed OSE Linux and civilization starter kit, packaged as a bootable virtual machine on PolymerOS: one image that carries the tools to build the house and the operating system to run on it.
